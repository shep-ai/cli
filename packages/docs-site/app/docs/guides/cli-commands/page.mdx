# CLI Commands

Complete reference for every `shep` command. Commands are grouped by function.

## Starting Shep

| Command        | Description                                               |
| -------------- | --------------------------------------------------------- |
| `shep`         | Start the Shep daemon. Equivalent to `shep start`.        |
| `shep start`   | Start the background service and open the dashboard.      |
| `shep stop`    | Stop the background service.                              |
| `shep restart` | Restart the background service.                           |
| `shep status`  | Print the current daemon status (running, stopped, port). |
| `shep ui`      | Open the web dashboard in your default browser.           |

```bash
# Start Shep in your project
cd ~/projects/my-app
shep

# Check if the daemon is running
shep status
```

## Features

Use `shep feat` to create, inspect, and manage features throughout their lifecycle.

### `shep feat new`

Create a new feature. Shep analyzes your repo, gathers requirements via AI, generates a plan, and starts implementation.

```bash
shep feat new
shep feat new --push --pr
shep feat new --fast
```

| Flag     | Description                                                           |
| -------- | --------------------------------------------------------------------- |
| `--push` | Push the feature branch to the remote after implementation completes. |
| `--pr`   | Open a pull request automatically (implies `--push`).                 |
| `--fast` | Skip detailed SDLC phases and go straight to coding.                  |

### `shep feat ls`

List all features in the current repository with their status.

```bash
shep feat ls
```

### `shep feat show <id>`

Display full details for a feature: status, requirements, plan, and agent activity.

```bash
shep feat show feat_abc123
```

### `shep feat del <id>`

Delete a feature and clean up its worktree and branch.

```bash
shep feat del feat_abc123
```

### `shep feat resume <id>`

Resume a paused or interrupted feature from where it left off.

```bash
shep feat resume feat_abc123
```

### `shep feat review <id>`

Start a review of the completed feature. Shep presents a summary of changes for your approval.

```bash
shep feat review feat_abc123
```

### `shep feat approve <id>`

Approve a feature after review. If `--push` or `--pr` flags were set, Shep proceeds with those actions.

```bash
shep feat approve feat_abc123
```

### `shep feat reject <id>`

Reject a feature after review. Shep pauses the feature and waits for further instructions.

```bash
shep feat reject feat_abc123
```

### `shep feat logs <id>`

Stream or display logs for a feature, including agent output and phase transitions.

```bash
shep feat logs feat_abc123
```

## Agents

Use `shep agent` to monitor and control the AI agents working on your features.

### `shep agent ls`

List all agents, both active and completed.

```bash
shep agent ls
```

### `shep agent show <id>`

Display details for a specific agent: assigned feature, current task, and output.

```bash
shep agent show agent_xyz789
```

### `shep agent stop <id>`

Stop a running agent. The agent finishes its current operation and exits.

```bash
shep agent stop agent_xyz789
```

### `shep agent logs <id>`

Stream or display the raw output from an agent.

```bash
shep agent logs agent_xyz789
```

### `shep agent delete <id>`

Delete an agent record and its associated resources.

```bash
shep agent delete agent_xyz789
```

### `shep agent approve <id>`

Approve a pending agent action (used when approval is required before each task).

```bash
shep agent approve agent_xyz789
```

### `shep agent reject <id>`

Reject a pending agent action. The agent skips the task and moves to the next one.

```bash
shep agent reject agent_xyz789
```

## Settings

Manage global and per-repo configuration. Running `shep settings` without a subcommand launches the interactive wizard.

| Command                  | Description                                          |
| ------------------------ | ---------------------------------------------------- |
| `shep settings`          | Launch the interactive settings wizard.              |
| `shep settings show`     | Print current settings to the terminal.              |
| `shep settings init`     | Run first-time setup (agent selection, auth, IDE).   |
| `shep settings agent`    | Configure which AI coding agent Shep uses.           |
| `shep settings ide`      | Set your preferred IDE (VS Code, Cursor, etc.).      |
| `shep settings workflow` | Configure workflow behavior (approval, auto-commit). |
| `shep settings model`    | Set AI model preferences for planning and analysis.  |

```bash
# Quick agent switch
shep settings agent

# View everything
shep settings show
```

## Tools

### `shep tools list`

List available tools and their install status.

```bash
shep tools list
```

### `shep install`

Install required tools and dependencies for Shep to function.

```bash
shep install
```

### `shep ide-open`

Open the current repository in your configured IDE.

```bash
shep ide-open
```

## Other Commands

### `shep run`

Run the configured agent directly on the current repository without the full feature workflow.

```bash
shep run
```

### `shep version`

Print the installed Shep version.

```bash
shep version
```

### `shep upgrade`

Upgrade Shep to the latest published version.

```bash
shep upgrade
```

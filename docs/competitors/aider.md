## What It Does

Aider lets you pair program with LLMs directly in the terminal. Its defining feature is git-first design — every edit becomes a proper git commit with a descriptive message. It was one of the first tools to treat the terminal as the natural home for AI-assisted coding.

### Key Features

- **39K GitHub stars, 4.1M installs** — Proven at scale
- **Git-first** — Every change is auto-staged and committed with a message
- **Multi-model** — Claude, GPT-4, Gemini, DeepSeek, local models
- **Context-aware** — Understands repository structure and file relationships
- **Interactive** — Conversational coding with the ability to add/remove files from context

## How Shep Compares

|                     | Aider                        | Shep                          |
| ------------------- | ---------------------------- | ----------------------------- |
| **Focus**           | Interactive pair programming | Full lifecycle orchestration  |
| **Workflow**        | Conversational (you direct)  | Automated (agent drives)      |
| **Requirements**    | You describe in chat         | AI-generated PRD              |
| **Planning**        | Implicit                     | Explicit, reviewable          |
| **Git integration** | Auto-commit per edit         | Worktree per feature          |
| **Parallel work**   | Single conversation          | Multiple features in parallel |
| **CI integration**  | Not included                 | Automatic fix loop            |
| **Dashboard**       | Terminal only                | Web dashboard                 |

### What We Respect

Aider pioneered the idea that git should be a first-class citizen in AI coding tools. The auto-commit approach means you always have a clean history and can roll back any AI change. Paul Gauthier's focus on making the tool genuinely useful over flashy set the standard for the space.

### Where Shep Differs

Aider is interactive — you're the driver, the AI is your pair. Shep is autonomous — you describe a feature, and the agent drives the entire lifecycle. Different tools for different moments: Aider when you want to think through code together, Shep when you want to delegate an entire feature.

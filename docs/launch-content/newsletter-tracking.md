## Status Key

- **Not Started**: Submission not yet sent
- **Submitted**: Pitch sent, awaiting response
- **Responded**: Editor replied, in conversation
- **Confirmed**: Feature scheduled for specific date
- **Published**: Newsletter issue shipped
- **Declined**: Not a fit / passed on feature
- **Follow-up**: Sent follow-up after no response

## Response Templates

### If Editor Requests More Info
```
Thanks for the interest! Happy to provide:

1. Technical writeup (architecture deep-dive)
2. Demo GIF (autonomous workflow in action)
3. Screenshots (web dashboard, TUI, CLI)
4. HN discussion link (strong traction, [X] points / [Y] comments)
5. Access to founding team for Q&A

What format works best for [Newsletter Name]?

Best,
[CMO Name]
```

### If Editor Declines
```
No problem! Thanks for considering it.

If you know of other newsletters or communities that might be a good fit for shep, I'd appreciate any suggestions.

Also, happy to stay in touch for future updates (we're building in public and shipping fast).

Best,
[CMO Name]
```

```
Still interested in featuring shep in [Newsletter Name]? Happy to provide any additional materials.

Thanks,
[CMO Name]
```

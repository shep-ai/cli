# Hacker News Launch Materials

Prep materials for the HN launch campaign (DEV-11).

## Files

| File | Purpose |
|------|---------|
| `show-hn-post.md` | Draft of the "Show HN" submission (title, body, timing, metrics) |
| `tweet-thread.md` | Twitter cross-post thread (use 3+ hours after HN submission if gaining traction) |
| `response-templates.md` | Quick reference for common questions/objections during engagement |
| `pre-flight-checklist.md` | Final verification before posting |

## Launch Timeline

### Phase 1: Pre-Flight (1-2 days before)
- [ ] Review all materials with CEO/board
- [ ] Confirm product is in optimal state (CTO sign-off)
- [ ] Test quick start flow (`npx @shepai/cli feat new`)
- [ ] Clear calendar for 4+ hours of engagement
- [ ] Draft tweet thread
- [ ] Set up metrics tracking (GitHub stars, HN score, referrer traffic)

### Phase 2: Launch Day
- [ ] Post on HN (Tuesday-Thursday, 8-10am PT)
- [ ] Notify team in CEO channel
- [ ] Monitor comments every 15-30 minutes (first 4 hours)
- [ ] Respond to every comment within 30-60 minutes
- [ ] Cross-post to Twitter after 3+ hours (if gaining traction)
- [ ] Offer video calls to top commenters

### Phase 3: Post-Launch (1-7 days after)
- [ ] Continue monitoring HN thread (diminishing engagement after 24h)
- [ ] Track GitHub star velocity (goal: 1K-3K in 48h)
- [ ] Write HN retrospective (transparent, data-driven)
- [ ] Share learnings with team
- [ ] Update growth plan based on performance
- [ ] Prepare next channel (Reddit or PH) based on HN results

## Success Metrics

**Tier 1 (Good):**
- 100+ points on HN
- Front page for 2+ hours
- 500-1K GitHub stars in 48h

**Tier 2 (Great):**
- 200+ points on HN
- Front page for 4+ hours
- 1K-2K stars in 48h

**Tier 3 (Exceptional):**
- 300+ points on HN
- Front page for 8+ hours
- 2K-3K+ stars in 48h

## Key Principles

1. **Technical depth over hype:** HN values substance. No marketing fluff.
2. **Transparency:** Acknowledge limitations honestly.
3. **Engagement:** Respond to every comment. Offer to pair with users who hit issues.
4. **Humility:** Don't oversell. Let the product speak for itself.
5. **Community-first:** Embrace feedback (positive or negative). Building in public.

## Coordination

**Before posting:**
- Get CTO sign-off on product readiness
- Notify CEO/board of timing
- Confirm README (DEV-4) and blockers (DEV-5) are done

**During launch:**
- Keep team updated on performance (CEO channel)
- Escalate urgent product issues to CTO
- Share standout community feedback

**After launch:**
- Debrief with team (what worked, what didn't)
- Update growth plan based on learnings
- Prepare next campaign (Reddit, PH, newsletters)

## Notes

- This is a **one-shot opportunity**. HN detects and kills reposts. If it flops, wait 6+ months before trying again.
- Don't ask friends to upvote. Don't use multiple accounts. HN detects this and shadowbans.
- Be patient. HN is unpredictable. Front page at 2pm can die by 4pm. Focus on engagement quality, not just score.
- The real goal is **long-term community building**, not just a one-day spike. Treat every commenter as a potential advocate.

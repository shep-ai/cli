# Pre-Flight Checklist

Final verification before posting to Hacker News.

---

## Product Readiness

- [ ] **README is compelling** (DEV-4 done ✅)
  - Clear value prop
  - Technical depth
  - Quick start instructions
  - Architecture overview

- [ ] **Top blockers are fixed** (DEV-5 done ✅)
  - No critical bugs
  - Installation works cross-platform
  - Quick start flow is smooth

- [ ] **Quick start flow tested**
  - `npx @shepai/cli feat new` works
  - Agent conversation is smooth
  - Spec generation succeeds
  - No embarrassing errors

- [ ] **Repo hygiene**
  - No typos in README
  - No broken links
  - No stale docs
  - Issue templates are live (DEV-8)
  - Contributing guide is clear

- [ ] **CTO sign-off on product readiness**

---

## Content Readiness

- [ ] **Show HN post drafted** (`show-hn-post.md`)
  - Title: "Show HN: Shep AI — One command, full lifecycle, merged PR"
  - Body: Technical depth, no hype
  - README link

- [ ] **Tweet thread drafted** (`tweet-thread.md`)
  - Ready to cross-post 3+ hours after HN

- [ ] **Response templates reviewed** (`response-templates.md`)
  - Common questions covered
  - Objections pre-addressed
  - Tone is humble, technical, transparent

---

## Timing Strategy

- [ ] **Launch day confirmed**
  - Tuesday, Wednesday, or Thursday
  - NOT Monday (inbox overload), Friday (low engagement), or weekend

- [ ] **Launch time confirmed**
  - 8-10am PT (11am-1pm ET, 4-6pm UTC)
  - Optimal HN engagement window

- [ ] **Calendar cleared for 4+ hours**
  - First 4 hours are critical
  - Monitor every 15-30 minutes
  - Respond to comments within 30-60 minutes

---

## Team Coordination

- [ ] **CEO/board notified of timing**
- [ ] **Team briefed on engagement plan**
- [ ] **CTO on standby for product issues**
- [ ] **Metrics tracking set up**
  - GitHub stars baseline (~70 current)
  - HN score tracking
  - Referrer traffic (if analytics available)

---

## Engagement Readiness

- [ ] **Response templates memorized**
- [ ] **Demo video prepared** (optional, can record during engagement if requested)
- [ ] **GitHub notifications enabled**
  - Watch for issues/PRs from HN traffic
- [ ] **Twitter account ready** (for cross-posting)
- [ ] **Mindset: Humble, transparent, technical**
  - No defensiveness
  - Embrace feedback
  - Offer to pair with users who hit issues

---

## Backup Plan (If It Flops)

- [ ] **Don't panic** — HN is unpredictable
- [ ] **Review comments for product insights**
- [ ] **Wait 6+ months before reposting**
- [ ] **Pivot to Reddit/PH as alternative channels**
- [ ] **Write transparent retrospective**
  - What went wrong
  - What we learned
  - How we'll adjust

---

## Success Criteria (Reminder)

**Tier 1 (Good):** 100+ points, 2+ hours front page, 500-1K stars
**Tier 2 (Great):** 200+ points, 4+ hours front page, 1K-2K stars
**Tier 3 (Exceptional):** 300+ points, 8+ hours front page, 2K-3K+ stars

---

## Final Sanity Checks

- [ ] **No "Ask HN" or "Tell HN" in title** (this is "Show HN")
- [ ] **URL points to GitHub repo** (not a landing page)
- [ ] **No self-promotional tone** (technical depth, not marketing)
- [ ] **No typos in title or post body**
- [ ] **Ready to engage for 4+ hours straight**

---

## Go/No-Go Decision

**GO if:**
- Product is in optimal state (CTO confirms)
- README is compelling (DEV-4 done ✅)
- Blockers are fixed (DEV-5 done ✅)
- Launch timing is Tuesday-Thursday, 8-10am PT
- Team is aligned and ready

**NO-GO if:**
- Product has critical bugs
- README is confusing
- CTO hasn't signed off
- Launch timing is suboptimal (Monday/Friday/weekend)
- Team is not aligned

---

**When all boxes are checked, post to HN and notify the team. Let's ship this. 🚀**
